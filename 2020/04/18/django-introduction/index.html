一、设计模式

1. MVC

一种软件设计典范，用一种业务逻辑、数据、界面显示分离的方法组织代码，将业务逻辑聚集到
一个部件里面，在改进和个性化定制界面及用户交互的同时，不需要重新编写业务逻辑。
MVC被独特的发展起来用于映射传统的输入、处理和输出功能在一个逻辑的图形化用户界面的结构中。
核心：解耦

编程模式：

  • Model (模型)

    • 是应用程序中用于处理应用程序数据逻辑的部分
    • 通常模型对象负责在数据库中存取数据

  • View (视图)

    • 是应用程序中处理数据显示的部分
    • 通常视图是依据模型数据创建的

  • Controller (控制器)

    • 是应用程序中处理用户交互的部分
    • 通常控制器负责从视图读取数据，控制用户输入，并向模型发送数据

优点: 降低各功能模块之间的耦合性，方便变更，更容易重构代码，最大程度上实现代码的重用

2. MTV

本质上与MVC模式没有什么差别，也是各组件之间为了保持松耦合关系，只是定义上有些许不同

编程模式

  • Model(模型)

    • 负责业务对象与数据库的对象(ORM)

  • Template(模版)

    • 负责如何把页面展示给用户

  • View(视图)

    • 负责业务逻辑，并在适当的时候调用Model和Template

注意: Django还有一个url分发器，它的作用是将一个个URL的页面请求分发给不同的view处理，view再调用相应的Model和Template

二、Django安装

环境：Mac os

1. pip 安装

pip install Django

2. 下载/git 安装

点击下载网址, 左侧有最新版

sudo python setup.py install

三、项目介绍

1. 新建项目

创建名为testproject的项目
注：所有路径不要有中文
终端输入

django-admin.py startproject testproject

2. 目录层级

testproject
├── manage.py        :一个命令行工具，可以让我们用多种方式对Django项目进行交互
└── testproject
    ├── __init__.py  :一个空文件，它告诉Python这个目录应该被看做一个包
    ├── asgi.py      :项目与ASGI兼容的Web服务器入口, 3.0以上版本特有
    ├── settings.py  :项目的配置文件（主要处理文件）
    ├── urls.py      :项目的url声明 （主要处理文件）
    └── wsgi.py      :项目与WSGI兼容的Web服务器入口

3. 启动服务

cd testproject # 切换到我们创建的项目
python manage.py runserver ip:port
……
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.

ip:port 不写时默认：http://127.0.0.1:8000/

4. 连接数据库

在settings.py 中配置
任选一个配置即可

  • Django 默认是sqlite数据库

  • 配置mongo数据库

Django 框架的数据库引擎中没有 MongoDB 引擎，配置文件 setting.py 中的 ENGINE 字段只支持常见几种关系型数据库。

  • django.db.backends.sqlite3
  • django.db.backends.mysql
  • django.db.backends.oracle
  • django.db.backends.postgresql
  • django.db.backends.postgresql_psycopg2

官方解释: 如果数据库使用 MongoDB，推荐使用 Djongo这个库来做数据库引擎

sudo pip install djongo

在settings.py 中配置

from mongoengine import *
database_name = 'test'
MONGO_DATABASE_OPTIONS = {
    'host': 'localhost',
    'port': 27017,
    # 'username': 'tor_tester',
    # 'password': '123456',
}
register_connection('default', database_name, **MONGO_DATABASE_OPTIONS)

DATABASES = {
    'default': {
        'ENGINE': 'djongo',
        'ENFORCE_SCHEMA': True,
        'NAME': 'test',
        'CLIENT': {
            'host': '127.0.0.1',
        }
    }
}

7. 定义模型

在models.py文件中定义模型, 一个模型对应一张表

ORM：Object Relational Mapping 对象关系映射

示例：

import mongoengine

class StudentModel(mongoengine.Document):
    name = mongoengine.StringField(max_length=16)
    age = mongoengine.IntField(default=0)

8. 创建视图

在views.py中定义视图

from django.shortcuts import render, HttpResponse

# Create your views here.
# .表示当前包下的models
from .models import StudentModel
from django.views.generic import View

class Student(View):
    def get(self, request):
        # 增加数据
        StudentModel.objects.create(name='张三', age= 18)
        # 查询数据
        result = StudentModel.objects.filter(name='张三')
        print(result[0].age)
        # # 修改数据
        # result = StudentModel.objects.filter(name='张三').first().update(name='李四')
        # print(result)
        # # 删除数据
        # result = StudentModel.objects.filter(name='张三').first().delete()
        # print(result)
        return HttpResponse(result[0].age)

9. 配置url

在urls.py中配置url

from django.urls import path, re_path
from django.contrib import admin
# import os
# print(os.getcwd())
from myapp.views import Student

urlpatterns = [
    re_path(r'^admin/', admin.site.urls),
    re_path(r'^student/$', Student.as_view(), name='student')
]

12. Admin站点管理

  • 概述：

    • 内容发布：负责添加，修改，删除内容的
    • 公告访问

  • 配置Admin应用：

    • 在 settings.py 文件中的 INSTALLED_APPS 中添加 ‘django.contrib.admin’, —-这条默认是添加好的。

  • 创建管理员用户：

    • 在项目目录下执行 python manage.py createsuperuser
    • 依次输入账号名，邮箱，密码即可完成用户创建

  • 登陆：
    http://127.0.0.1:8000/admin/

  • 汉化(可选)：
    把 project/settings.py 中作如下设定：
    LANGUAGE_CODE = ‘zh-Hans’
    TIME_ZONE = ‘Asia/Shanghai’

  • 管理数据表：
    修改 myAPP/admin.py 如下：

from django.contrib import admin
# Register your models here.
from .models import StudentModel
# 注册
admin.site.register(StudentModel)

注意：使用MongoDB会报错，暂未解决

13. 模板配置

两种

  • 在App中进行模板配置

    • 只需在App的根目录创建templates文件夹即可
    • 如果想让代码自动提示，我们应该标记文件夹为模板文件夹

  • 在项目目录中进行模板配置

    • 需要在项目目录中创建templates文件夹并标记
    • 需要在settings中进行注册
    • 在开发中使用第二种
    • 模板可以继承，复用
