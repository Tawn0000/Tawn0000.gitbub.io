一、安装JDK

JDK全称Java SE Development kit(JDK)，即java开发工具包，是Oracle提供的一套用于开发java应用程序的开发包，它提供编译，运行java程序所需要的各种工具和资源，包括java编译器，java运行时环境，以及常用的java类库等。

JRE Java Runtime Environment(JRE) Java运行环境

JVM Java Virtual Machine Java虚拟机，JRE包含JVM。JVM是运行java程序的核心虚拟机，还需要其他的类加载器，字节码校验器，以及大量的基础类库。JRM除了包含JVM之外，还包含运行java程序的其它环境支持。

二、安装Scala

Scala 是一门多范式（multi-paradigm）的编程语言，设计初衷是要集成面向对象编程和函数式编程的各种特性。
Scala 运行在Java虚拟机上，并兼容现有的Java程序。
Scala 源代码被编译成Java字节码，所以它可以运行于JVM之上，并可以调用现有的Java类库。
所以请预先安装好Java环境

1. brew安装

有多种方法安装，首先尝试一下 brew 安装，如果没有安装brew, 参考安装brew
打开Mac终端输入

$ brew install scala

如果一直停在 Updating Homebrew…, 可以使用Control + C 暂时停止其更新
注意：brew安装后会自动设置环境变量，所以不用再手动设置环境变量 ->Homebrew安装Packages是否需要另外设置环境变量PATH?

2. 手动安装

如果brew 安装不上可以手动下载安装
Scala官网 下载

  • 下载 Scala 二进制包解压缩文件包然后移动到自己指定的安装位置
  • 配置环境变量 在 ~/.bash_profile 中加入下列路径（根据自己的安装路径修改）步骤同上, 最后不能忘记终端运行 source ~/.bash_profile
export PATH="$PATH:/usr/local/share/scala/bin"

3. 检查安装

  • 检查版本

终端输入：

$ scala -version

输出：

Scala code runner version 2.13.1 -- Copyright 2002-2019, LAMP/EPFL and Lightbend, Inc.
  • 测试环境：

终端输入：

$ scala

输出：

Welcome to Scala 2.13.1 (Java HotSpot(TM) 64-Bit Server VM, Java 1.8.0_241).
Type in expressions for evaluation. Or try :help.

三、配置sshd

  • sshd为secure shell的简称；SSH为建立在应用层基础上的安全协议。SSH是目前较可靠，专为远程登录会话和其他网络服务提供安全性的协议
  • 连接方式（在客户端）：
    • ssh username@ip #文本模式
    • ssh -X username@ip #可以在链接成功后开启图形界面
  • 注意：
    • 第一次链接陌生主机是要建立认证文件,然后会询问是否建立，需要输入yes
    • 再次链接此台主机时，因为已经生成~/.ssh/know_hosts文件所以不需要再次输入yes
  • 远程复制：格式
    • scp file root@id:dir(文件的上传)
    • scp root@if:/dir file(文件的下载)

在mac终端依次执行以下命令

$ ssh-keygen -t dsa -P '' -f ~/.ssh/id_dsa
$ cat ~/.ssh/id_dsa.pub >> ~/.ssh/authorized_keys
mac 启动sshd
$ sudo launchctl load -w /System/Library/LaunchDaemons/ssh.plist
查看启动
$ sudo launchctl list | grep ssh
输出- 0 com.openssh.sshd 表示启动成功
停止sshd服务
$ sudo launchctl unload -w /System/Library/LaunchDaemons/ssh.plist

测试ssh免密码登录

终端输入：

$ ssh localhost

输出：

Last login: Thu Mar 26 12:00:09 2020

如果输出：

ssh: connect to host localhost port 22: Connection refused

则需要打开mac远程登陆功能： System Preferences -> Sharing -> 勾选Remote Login

解释：

  • id_rsa.pub和id_dsa.pub有什么区别？
    id_rsa.pub和id_dsa.pub是id_rsa和id_dsa的。如果您要求与SSH相关，id_rsa是 RSA 密钥，可与SSH一起使用协议1或2，而id_dsa是 DSA 密钥，只能与SSH协议2一起使用。两者都非常安全，但DSA似乎是标准的（假设所有的客户端/服务器支持SSH 2）。
    参见：id_rsa.pub和id_dsa.pub有什么区别？
  • launchctl 命令的作用
    使sshd开机启动
    参见：
    Mac中的定时任务利器：launchctl
    OS X 添加定时任务
  • ssh 和 sshd
    当主机中开启openssh服务，那么就对外开放了远程连接的接口
    ssh为openssh服务的客户端，sshd为openssh服务的服务端
    远程管理工具ssh具有数据加密传输、网络开销小以及应用平台范围广的特点，是远程管理中最常见的控制工具
    参见：Linux系统的ssh与sshd服务

四、安装Hadoop

Hadoop是一个由Apache基金会所开发的分布式系统基础架构。用户可以在不了解分布式底层细节的情况下，开发分布式程序。充分利用集群的威力进行高速运算和存储。
Hadoop实现了一个分布式文件系统（Hadoop Distributed File System），简称HDFS。HDFS有高容错性的特点，并且设计用来部署在低廉的（low-cost）硬件上；而且它提供高吞吐量（high throughput）来访问应用程序的数据，适合那些有着超大数据集（large data set）的应用程序。HDFS放宽了（relax）POSIX的要求，可以以流的形式访问（streaming access）文件系统中的数据。
Hadoop的框架最核心的设计就是：HDFS和MapReduce。HDFS为海量的数据提供了存储，而MapReduce则为海量的数据提供了计算

Spark 有预装hadoop的包和不预装的包， 如果安装spark可以先跳过hadoop的安装试一下

1. brew安装

如果没有安装brew, 参考安装brew
终端输入：

brew install Hadoop

brew 安装不需要手动配置环境，同上。

2. 手动安装

brew安装不成功可以采用手动安装
官网下载安装包
binary是编译好的可以直接使用，source是还没编译过的源代码，需要自行编译。这里我们下载binary就好。

配置环境变量

打开Mac终端输入

$ vim ~/.bash_profile

在其中输入，退出保存

$ export HADOOP_HOME= 安装的位置，如: /usr/local/Cellar/hadoop/3.2.1_1
$ export PATH=$PATH:$HADOOP_HOME/bin

终端输入

$ source ~/.bash_profile

使环境配置生效

3.Hadoop配置

/usr/local/Cellar/hadoop/3.2.1_1/libexec/etc/hadoop 目录下添加下列内容

  • core-site.xml
<configuration>
  <property>
     <name>hadoop.tmp.dir</name>
     <value>file:/usr/local/Cellar/hadoop/3.2.1.1/libexec/tmp</value>
  </property>
  <property>
     <name>fs.defaultFS</name>
     <value>hdfs://localhost:9000</value>
  </property>
</configuration>
  • hadoop-env.sh
# 配置Hadoop使用的JDK版本，注意不要使用jdk1.9, 高版本会有问题
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.8.0_241.jdk/Contents/Home

 # 增加导出
export HADOOP_OPTS="-Djava.library.path=${HADOOP_HOME}/lib/native"
  • hdfs-site.xml
<configuration>
    <property>
        <name>dfs.permissions.enabled</name>
        <value>false</value>
    </property>
    <property>
         <name>dfs.replication</name>
         <value>1</value>
  </property>
    <property>
                <name>dfs.http.address</name>
                <value>localhost:50070</value>
    </property>

  <property>
         <name>dfs.namenode.name.dir</name>
         <value>file:/usr/local/Cellar/hadoop/3.2.1.1/libexec/tmp/dfs/name</value>
  </property>
  <property>
         <name>dfs.namenode.data.dir</name>
         <value>file:/usr/local/Cellar/hadoop/3.2.1.1/libexec/tmp/dfs/data</value>
  </property>
</configuration>
  • mapred-site.xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
</configuration>
  • yarn-site.xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
</configuration>

五、安装Spark&pyspark

1.brew安装

如果没有安装brew, 参考安装brew

终端输入

$ brew install apache-spark
# 检测是否安装成功，启动spark
cd /usr/local/Cellar/apache-spark/1.3.0/bin/
./spark-shell
# 或者直接输入
spark-shell

2. 手动安装

如果brew安装不了，可以手动安装

  • 官网下载安装包,传送门
    根据是否预装了hadoop选择相应版本 参见：Spark 下载版本间的区别

  • 解压缩安装包到/usr/local下
    如：$sudo tar -zxf ~/Dowmloads/spark-2.0.2-bin-hadoop2.7.tgz -C /usr/local/$
    可以更改一下文件名 $sudo mv ./spark-2.0.2-bin-hadoop2.7 ./spark$

  • 配置环境变量

打开Mac终端输入

$ vim ~/.bash_profile

在其中输入，退出保存

$ export SPARK_HOME=/usr/local/spark
$ export PATH=$PATH:$HADOOP_HOME/bin

终端输入

$ source ~/.bash_profile

使环境配置生效

3. Spark配置

# 打开自己的安装路径，如果是brew安装的应该会在/usr/local/Cellar/下
$ cd /usr/local/Cellar/apache-spark/2.4.5/libexec/conf
$ cp spark-env.sh.template spark-env.sh
$ vim spark-env.sh
# 添加如下内容
# 根据自己的安装路径更改
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.8.0_241.jdk/Contents/Home
export SCALA_HOME=/usr/local/Cellar/scala/2.13.1_1
export HADOOP_CONF_DIR=/usr/local/Cellar/hadoop/3.2.1_1/libexec/etc/hadoop
export SPARK_MASTER_IP=localhost
export SPARK_LOCAL_IP=localhost
export SPARK_WORKER_CORES=2
export SPARK_WORKER_MEMORY=2g

$ cp slaves.template slaves

5. 安装pyspark

mac 终端输入

$ pip install --upgrade pyspark -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host mirrors.aliyun.com

输出

Looking in indexes: https://pypi.tuna.tsinghua.edu.cn/simple
Requirement already up-to-date: pyspark in /Users/tawn/.pyenv/versions/3.6.8/lib/python3.6/site-packages (2.4.5)
Requirement already satisfied, skipping upgrade: py4j==0.10.7 in /Users/tawn/.pyenv/versions/3.6.8/lib/python3.6/site-packages (from pyspark) (0.10.7)

如果报错：ERROR: Command errored out with exit status 1: python setup.py egg_info Check the logs for full command output.

$ sudo pip install --upgrade pip
$ sudo pip install --upgrade setuptools

如果报错：ERROR: Could not find a version that satisfies the requirement pypandoc

$ pip install pypandoc
$ pip install --upgrade pyspark -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host mirrors.aliyun.com

6. pyspark 启动jupyter

在 .bash_profile 中加入以下内容

export SPARK_HOME=/usr/local/Cellar/apache-spark/2.4.5/libexec
export PATH=$SPARK_HOME/bin:$PATH

export PYSPARK_DRIVER_PYTHON=jupyter
export PYSPARK_DRIVER_PYTHON_OPTS="notebook"

终端输入

$ source ~/.bash_profile

使环境配置生效

demo

# $ pyspark
# 会自动启动jupyter notebook
# 新建一个交互脚本，输入
myRDD = sc.parallelize([('Rose', 19), ['joey', 18]])
myRDD.take(2)
#输出
[('Rose', 19), ['joey', 18]]
